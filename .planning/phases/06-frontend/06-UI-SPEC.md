## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, inline label gaps |
| sm | 8px | Compact element spacing, panel row padding |
| md | 16px | Default element spacing, card padding, form field gap |
| lg | 24px | Section padding, panel header padding |
| xl | 32px | Layout gaps, page section breaks |
| 2xl | 48px | Major section breaks, hero spacing |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Graph canvas: no padding constraints — React Flow manages its own viewport
- Detail panel width: 380px fixed (D-06, locked)
- Course root node: 60px diameter (D-03, locked)
- Concept node diameter: 32px–48px linear by source_count (D-02, locked)
- Flashcard/quiz node diameter: 40px fixed (visually between concept and course root)
- Touch targets on interactive node overlays: minimum 44px (accessibility)

### 60/30/10 Contract

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1F1E1B` | Page background, graph canvas background, all major surfaces |
| Secondary (30%) | `#14120F` | Cards (dashboard course cards), detail panel background, nav sidebar, library table rows |
| Accent (10%) | `#C96442` terracotta | See reserved list below |
| Destructive | `#B5604A` | Error status badges, error state text, error toast only |

**Accent `#C96442` reserved for:**
1. Primary CTA buttons ("Create Course", "Generate Quiz", "Start Quiz")
2. "Open Cortex" link in notch footer (brand continuity)
3. Active state indicator on nav links
4. Course root node fill (60px anchor node)

**Amber `#C18A3F` reserved for:**
1. Gotcha highlight background in concept detail panel (text highlight, not full block)
2. Processing / sending status badge background
3. Source count dot / progress indicator while pipeline runs

**Moss `#6B8E5A` reserved for:**
1. Done / success status badge background
2. "View Flashcards" button (secondary action)
3. Flashcard node fill color (visually distinct from concept nodes per D-05)

**Struggle signal red ring:**
- Ring color: `#EF4444` (Tailwind red-500) — brighter than destructive `#B5604A` to be instantly visible against dark bg
- Ring implementation: CSS `box-shadow: 0 0 0 3px #EF4444` with `@keyframes cortex-pulse` animating opacity 1 → 0.3 → 1 at 1.5s ease-in-out infinite
- No filled color change on the node itself (D-04, locked)

## Interaction Contracts

### Graph Page (`/courses/[id]`)

**Node hover:**
- Tooltip appears after 300ms delay with concept title + source count (e.g. "Gradient Descent · 3 sources")
- Cursor changes to `pointer`

**Node click (concept):**
- Opens shadcn `Sheet` from the right at 380px width (D-06)
- Graph remains fully interactive (pan, zoom) while panel is open
- Panel has `×` close button top-right; clicking graph canvas outside panel does NOT close it
- URL stays at `/courses/[id]` — no route change

**Panel — "View Flashcards" click:**
- Panel content transitions in-place to flashcard flip view (D-07)
- Transition: 150ms fade between content states (no slide)
- Back button (`← Back to [concept title]`) restores concept detail in-place
- No navigation, no URL change

**Quiz node click:**
- Navigate to `/quiz/[id]` (D-08)
- Full page navigation — no Sheet

**Graph polling:**
- `refreshInterval: hasPendingSources ? 5000 : 0` (D-10, exact SWR pattern)
- When polling is active: subtle pulsing dot indicator bottom-left of graph ("Updating…")
- When polling stops: indicator disappears without fanfare

**Dagre layout:**
- Computed once on first data load; skip recalculation if node IDs unchanged (prevents viewport jumps)
- `setTimeout(() => reactFlowInstance.fitView(), 0)` after first `setNodes(layoutedNodes)`
- `import * as dagre from '@dagrejs/dagre'` exact import form (ROADMAP constraint)

### Flashcard Flip Mode

**Card face:**
- Front: question text, centered, 24px display weight, on card face
- Back: answer text, centered, 14px body weight, on reverse face
- Card dimensions: 480px × 280px, `border-radius: 12px`, border `rgba(255,255,255,0.12)` 1px
- Flip animation: CSS 3D transform `rotateY(180deg)` on `transform-style: preserve-3d`, 400ms ease-in-out
- "Show Answer" button below card — hidden after flip; replaced by "Next" button

**No grading UI.** No "Again / Hard / Good / Easy" buttons. Flip is the only action.

### Quiz Page (`/quiz/[id]`)

**Layout:** Single-column centered, max-width 640px
**Progress bar:** shadcn `Progress` component, top of page, updates per question
**MCQ:** shadcn `RadioGroup` with option labels; "Submit Answer" button disabled until selection
**Short answer:** `Textarea` (min 2 rows); "Submit Answer" enabled after ≥ 1 character
**Feedback display:** After submit — correct/incorrect badge + 1–2 sentence feedback from backend; "Next Question" button appears
**Final screen:** Score (e.g. "5 / 7 correct"), list of concepts to review as Badge links back to `/courses/[id]`

### Library Page (`/library`)

**Table columns:** Title, Type (badge), Course, Status (badge), Uploaded (relative time)
**Status polling:** Library page does NOT poll — static fetch, manual refresh button
**Uploader:** shadcn `Card` at bottom with file input, labeled: "Fallback uploader — prefer dropping into the notch"
**Course assignment dropdown:** shadcn `Select` component, populated from `GET /courses`

## Data Fetching Patterns

| Hook | Endpoint | Polling |
|------|----------|---------|
| `useCourses()` | `GET /courses` | No |
| `useCourseGraph(id)` | `GET /courses/{id}/graph` | `hasPendingSources ? 5000 : 0` |
| `useConceptDetail(id)` | `GET /concepts/{id}` | No |
| `useSources()` | `GET /sources` (library page) | No |
| `useQuiz(id)` | `GET /quiz/{id}` | No |

All hooks use SWR. `mutate` called after POST /ingest and POST /quiz to invalidate relevant caches.

## Accessibility

- All interactive nodes have `role="button"` and `aria-label="{concept title}"`
- Panel opens with focus trapped inside Sheet per Radix UI default
- Keyboard: `Escape` closes Sheet; `Space`/`Enter` on focused node opens Sheet
- Color is never the sole differentiator — edge types use both color AND stroke pattern (D-04 pulsing ring supplements, not replaces, contrast)
- Status badges have text labels (not icon-only)
