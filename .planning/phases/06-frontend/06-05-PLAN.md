<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Library uploader → POST /ingest | File upload with course_id and kind; backend validates all |
| File input accept attribute | Frontend accept=".pdf,.png,.jpg,.jpeg,.txt,.md" is UX only — backend validates actual content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05-01 | Input Validation | File upload — frontend accept attribute | mitigate | Frontend `accept` attribute provides UX feedback but is NOT a security control. Backend (ingest.py) validates file type and size (50MB limit from PIPE-01). Frontend should NOT rely on accept for security. |
| T-06-05-02 | Input Validation | File upload — kind field | accept | `kind` is a string sent to backend; backend validates it as a Literal["pdf","image","text","url"] via Pydantic — invalid values return 422. Frontend selector limits options but not security-critical. |
| T-06-05-03 | Information Disclosure | GET /sources — returns all sources with titles | accept | Titles are file names / URL titles supplied by the user themselves; no PII; single-user system |
| T-06-05-04 | Denial of Service | Library page static fetch (no polling) | accept | Single fetch on page load; no polling; no loop risk. Manual refresh via mutate only. |
</threat_model>
