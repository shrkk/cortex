  <name>Task 2: Smoke-test all 7 GRAPH endpoints against the live API</name>
  <what-built>
    Plans 02 and 03 implemented all 7 GRAPH requirements:
    - GRAPH-01: GET /courses — list courses
    - GRAPH-02: POST /courses — create course with id
    - GRAPH-03: GET /courses/{id}/graph — nodes + edges payload
    - GRAPH-04: GET /concepts/{id} — detail with summary field
    - GRAPH-05: node types course, concept, flashcard, quiz
    - GRAPH-06: graph endpoint performance (no N+1)
    - GRAPH-07: GET /courses/match — embed + cosine match
  </what-built>
  <how-to-verify>
    Prerequisites: Run `docker compose up -d && cd backend && uvicorn app.main:app --reload` (or however the API is started in this project).

    Run these curl commands and verify each expected output:

    **GRAPH-01: List courses**
    ```bash
    curl -s http://localhost:8000/courses | jq 'length'
    ```
    Expected: A number (0 or more). Must not error.

    **GRAPH-02: Create course**
    ```bash
    curl -s -X POST http://localhost:8000/courses \
      -H "Content-Type: application/json" \
      -d '{"title": "Graph API Test Course", "user_id": 1}' | jq '{id, title}'
    ```
    Expected: `{"id": <number>, "title": "Graph API Test Course"}` — id must be present and non-null.

    **GRAPH-03 + GRAPH-05: Graph endpoint node types** (use the course id from GRAPH-02 above, or course id=1 if seed data is loaded):
    ```bash
    curl -s "http://localhost:8000/courses/1/graph" | jq '{
      node_count: (.nodes | length),
      node_types: ([.nodes[].type] | unique | sort),
      edge_types: ([.edges[].type] | unique | sort)
    }'
    ```
    Expected: node_types includes "course" and "concept". edge_types includes "contains". No "embedding" key anywhere in the response.

    ```bash
    curl -s "http://localhost:8000/courses/1/graph" | jq '[.nodes[].id]' | grep -v '"course-\|concept-\|flashcard-\|quiz-'
    ```
    Expected: No output (all node ids are prefixed strings).

    **GRAPH-06: Performance** (requires seeded DB with 10+ concepts):
    ```bash
    time curl -s "http://localhost:8000/courses/1/graph" > /dev/null
    ```
    Expected: `real` time under 0.2s (200ms). Note: first call may be slower due to cold start; measure second call.

    **GRAPH-04: Concept detail — summary field**
    First get a concept id: `curl -s "http://localhost:8000/courses/1/graph" | jq '[.nodes[] | select(.type=="concept")][0].data.concept_id'`
    Then:
    ```bash
    curl -s "http://localhost:8000/concepts/<concept_id>" | jq 'keys | sort'
    ```
    Expected: keys include "id", "course_id", "title", "summary", "key_points", "gotchas", "examples", "student_questions", "source_citations", "flashcard_count", "struggle_signals". Key "definition" must NOT be present.

    ```bash
    curl -s "http://localhost:8000/concepts/<concept_id>" | jq '.summary'
    ```
    Expected: A string (not null) if the concept has a definition. May be null only if concept.definition is NULL in the DB.

    **GRAPH-07: Course match**
    ```bash
    curl -s "http://localhost:8000/courses/match?hint=backpropagation" | jq .
    ```
    Expected (if seed data loaded with CS229 course): `{"course_id": <n>, "title": "...", "confidence": <float >= 0.65}`. If no seed data, may return null — that is acceptable.

    ```bash
    curl -s "http://localhost:8000/courses/match?hint=zzzznonexistentzzz" | jq .
    ```
    Expected: `null`

    **404 guard:**
    ```bash
    curl -s "http://localhost:8000/courses/99999/graph" | jq '.detail'
    ```
    Expected: `"Course not found"`

    ```bash
    curl -s "http://localhost:8000/concepts/99999" | jq '.detail'
    ```
    Expected: `"Concept not found"`
