# Phase 2 — UI Design Contract

> Visual and interaction contract for the Swift/SwiftUI macOS notch surfaces.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform:** Swift/SwiftUI macOS — NOT web. shadcn does not apply.
> **Design system:** None (Swift/SwiftUI native components + macOS HIG).
> **Parent visual context:** NotchDrop fork — compact, dark, always-on-top overlay in the macOS notch.

## Spacing Scale

Base grid: 8pt (SwiftUI standard, matches macOS HIG).

All spacing values are multiples of 4pt.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon-to-label gap, tight inline padding |
| sm | 8pt | Row inner padding, button horizontal inset |
| md | 12pt | Default between stacked rows in CortexCourseTab |
| lg | 16pt | Section separation inside the notch panel |
| xl | 20pt | Outer horizontal padding of the notch expanded area |

Exceptions:
- Status pill horizontal padding: 10pt each side (pill shape requires non-standard inset — use `.padding(.horizontal, 10)`)
- Status pill vertical padding: 5pt each side (`.padding(.vertical, 5)`)
- Touch targets: minimum 44pt height for all tappable rows in CortexCourseTab (macOS HIG minimum)

---

## Typography

Font: SF Pro via `.font(.system(...))`. Never hardcode point sizes as raw magic numbers — use the table below.

| Role | SwiftUI Call | Size | Weight | Line Height |
|------|-------------|------|--------|-------------|
| Body (course row label) | `.font(.system(size: 13, weight: .regular))` | 13pt | Regular (400) | 1.4 (system default) |
| Label (secondary/hint text) | `.font(.system(size: 11, weight: .regular))` | 11pt | Regular (400) | 1.3 |
| Heading (tab/section title) | `.font(.system(size: 13, weight: .semibold))` | 13pt | Semibold (600) | 1.2 |
| Pill text | `.font(.system(size: 11, weight: .medium))` | 11pt | Medium (500) | 1.0 (single line only) |

Two weights only: Regular (400) for body/label, Semibold (600) for headings. Pill uses Medium (500) as a single exception for legibility at small size on dark background.

---

## Color

The notch environment is always dark. All surfaces render on a near-black blurred background. macOS semantic colors are preferred — they adapt automatically and satisfy accessibility contrast.

| Role | Value | SwiftUI Color | Usage |
|------|-------|--------------|-------|
| Dominant (60%) | Near-black translucent | `.black.opacity(0.85)` + `.ultraThinMaterial` on panel background | Notch expanded panel background |
| Secondary (30%) | Dark mid-gray | `Color(nsColor: .controlBackgroundColor)` or `Color.white.opacity(0.07)` | Course row background, settings field backgrounds |
| Accent (10%) | Cortex Indigo `#6366F1` | `Color(red: 0.388, green: 0.400, blue: 0.945)` | **Reserved for:** confirm/upload button fill, active course row border, "Sending…" pill background tint |
| Destructive | macOS Red | `Color(nsColor: .systemRed)` | Error pill background tint only |
| Success tint | macOS Green | `Color(nsColor: .systemGreen).opacity(0.15)` | "Sent to [Course]" pill background tint (subtle, not full fill) |

Accent reserved for: CTA button fill ("Confirm", "Upload"), selected/active course row left-border indicator, status pill background when in `sending` state.

Do NOT use accent on: body text, secondary labels, row backgrounds, separator lines.

Text on dark backgrounds: use `Color.white` with opacity modifiers:
- Primary text: `Color.white.opacity(0.90)`
- Secondary/hint text: `Color.white.opacity(0.55)`
- Disabled text: `Color.white.opacity(0.30)`

## Component Specs

### 1. Status Pill (CortexStatusPill)

Location: Overlay rendered on the notch panel, top-center. Does not block the course tab content.

States:

| State | Background | Icon | Copy | Duration |
|-------|-----------|------|------|----------|
| Sending | Accent `#6366F1` at 85% opacity | SF Symbol `arrow.up.circle` | "Sending to Cortex…" | Persistent until response |
| Success | `systemGreen` at 15% opacity, white border 0.5pt | SF Symbol `checkmark.circle.fill` | "Sent to [Course Name]" | Auto-dismiss after 2s |
| Error | `systemRed` at 85% opacity | SF Symbol `exclamationmark.circle.fill` | "Upload failed — check settings" | Auto-dismiss after 4s |

Geometry:
- Height: 26pt
- Corner radius: 13pt (fully rounded pill)
- Horizontal padding: 10pt, vertical: 5pt
- Icon size: 12pt, gap to text: 4pt
- Entry animation: `.spring(response: 0.3, dampingFraction: 0.7)` slide-down from top + fade-in
- Exit animation: `.easeOut(duration: 0.25)` fade-out

### 2. CortexCourseTab

Location: Slides inline into the expanded notch panel (not a separate window or popover). Height is dynamic — minimum 120pt, maximum 280pt before scrolling.

Visual geometry:
- Width: fills available notch panel width minus 2× outer horizontal padding (20pt each side)
- Corner radius: 10pt
- Background: secondary color (`Color.white.opacity(0.07)`) over the panel's `.ultraThinMaterial`
- Inner padding: 12pt all sides

Sub-states:

**State A — No courses exist (ING-11):**
- Heading: "Name this course" (13pt semibold)
- Text field: placeholder "e.g. CS 229: Machine Learning" (11pt regular, secondary opacity)
- TextField background: `Color.white.opacity(0.10)`, corner radius 6pt, height 32pt
- CTA button: "Confirm" — accent fill, white text, 13pt semibold, height 32pt, corner radius 6pt
- Button disabled state: accent at 40% opacity when text field is empty

**State B — Courses exist, picker shown (ING-13):**
- Heading: "Send to…" (13pt semibold)
- Course list rows: 40pt height each, 12pt horizontal padding, 8pt gap between rows
- Row selected indicator: 2pt left border in accent color, row background `Color.white.opacity(0.12)`
- Row unselected: `Color.white.opacity(0.04)` background
- "New course…" option: last row, label in secondary opacity, SF Symbol `plus.circle` leading icon at 12pt
- Pre-selected row (ING-15): last-selected course ID from `UserDefaults` key `cortex.lastCourseId` pre-highlighted on appearance
- Scroll enabled when list exceeds 4 rows

**State C — Auto-assigned (ING-12):**
- CortexCourseTab does NOT appear. Status pill goes directly to "Sending" state.
- No UI shown to user. Silent assignment.

Transition animation: `.easeInOut(duration: 0.2)` for tab slide-in/out within the notch panel.

### 3. CortexSettings Panel

Location: Accessible from a settings gear icon in the notch expanded state.

Layout: `VStack(alignment: .leading, spacing: 16)` with 20pt outer padding.

Controls:

| Control | Label | Type | Default |
|---------|-------|------|---------|
| Enable Cortex | "Enable Cortex Drop" | `Toggle` (macOS native) | on |
| Backend URL | "Backend URL" | `TextField` (bound to `UserDefaults` key `cortex.backendURL`) | `http://localhost:8000` |

- Toggle: macOS `Toggle` style (system checkbox on macOS) — do NOT use custom toggle
- TextField: 11pt regular, `Color.white.opacity(0.10)` background, 28pt height, 6pt corner radius
- Label text: 13pt semibold, `Color.white.opacity(0.90)`
- Helper text below Backend URL field: "Default: http://localhost:8000" in 11pt regular at 55% opacity
- When Cortex is disabled (ING-09): all Cortex logic bypassed; notch falls back to original NotchDrop behavior; settings panel remains accessible so user can re-enable

## Interaction Contracts

### Drag-and-Drop Entry Flow

```
User drags item onto notch
        ↓
NSItemProvider priority check:
  1. public.png / public.jpeg (image UTI) — checked FIRST
  2. public.file-url (file URL) — checked SECOND
  (See ROADMAP stack note — UTI ordering is critical)
        ↓
#if CORTEX_ENABLED guard → true?
        ↓ yes
GET /courses/match?hint=[filename or content preview]
        ↓
confidence ≥ 0.65?
  → YES: auto-assign silently, show "Sending" pill, POST /ingest
  → NO (or null): show CortexCourseTab picker (State B) or creator (State A)
        ↓ (after user confirms course)
Show "Sending to Cortex…" pill
POST /ingest → {source_id, status: "pending"}
        ↓
Response received within 200ms?
  → SUCCESS: pill transitions to "Sent to [Course Name]", auto-dismiss 2s
  → ERROR: pill transitions to error state, auto-dismiss 4s
```

### ⌘V Paste Flow

Same as drag-and-drop flow after clipboard inspection (ING-04, ING-05, ING-06). Source type determined by clipboard UTI.

### Session Default Pre-selection (ING-15)

On every CortexCourseTab appearance:
1. Read `UserDefaults.standard.integer(forKey: "cortex.lastCourseId")`
2. If value > 0 and a matching course row exists in the list → scroll to it, apply selected visual state
3. On user confirmation → write the confirmed course ID back to `UserDefaults`
4. Default is overridable — user can tap any other row

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (new course) | "Confirm" |
| Primary CTA (existing course) | "Send to [Course Name]" (course name interpolated at runtime) |
| Status: sending | "Sending to Cortex…" |
| Status: success | "Sent to [Course Name]" (course name interpolated) |
| Status: error | "Upload failed — check settings" |
| Empty state: no courses | "Name this course" (heading) + placeholder "e.g. CS 229: Machine Learning" |
| Settings toggle label | "Enable Cortex Drop" |
| Settings URL label | "Backend URL" |
| Settings URL helper | "Default: http://localhost:8000" |
| New course option in picker | "New course…" |
| Destructive action | None in Phase 2 — no delete or destructive actions exist |

Rules:
- Course names render verbatim from the API — no truncation in the status pill (max expected: ~40 chars)
- "Cortex" is always capitalized as a proper noun
- No exclamation marks anywhere
- No sentence-final periods in pill/label copy (these are UI labels, not sentences)

## Accessibility

- Minimum touch/click target: 44pt (macOS HIG)
- All SF Symbols paired with visible text labels (no icon-only tappable elements except the settings gear)
- Settings gear: `.accessibilityLabel("Cortex settings")`
- Status pill: `.accessibilityLabel("Cortex status: [state copy]")` — announced on state transition
- Color contrast: white text at 90% opacity on near-black background satisfies WCAG AA (contrast ratio > 7:1)
- Reduced motion: check `NSAccessibility.isReduceMotionEnabled`; if true, replace spring animations with instant transitions (`.animation(nil)`)
